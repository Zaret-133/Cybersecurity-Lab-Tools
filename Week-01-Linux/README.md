# Week 01 — Linux Fundamentals

## 📌 Overview

This week focused on mastering Linux command-line navigation, file system structure, permissions, and basic log analysis.

---

## 🎯 Lab Objective

Build a strong Linux foundation for cybersecurity by learning how to navigate systems, manage permissions, review logs, and identify security risks.

## 🧩 Lab Components

| Step | Focus |
| :--- | :--- |
| 🐧 Linux | File system navigation and FHS |
| 🔐 Permissions | `chmod`, `chown`, and access control |

## 🧠 Lab Flow

```text
Linux Navigation → File Permissions → Log Analysis → Threat Extraction → Final Report
```

---

## 🛠️ Skills Developed

- Linux filesystem navigation using the Filesystem Hierarchy Standard (FHS)
- File and directory permissions using `chmod` and `chown`
- Log file analysis using `grep`, `awk`, and `sort`
- Identifying system-level security risks
- Organizing findings into a clean security report

## 🔐 Security Insight

Improper file permissions can expose sensitive data, allow unauthorized access, or create opportunities for privilege escalation.

Log analysis is also a key Blue Team skill because it helps identify suspicious activity, failed access attempts, and possible indicators of compromise.
